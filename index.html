What Epoch is

In one sentence

Epoch is an interactive civilization-flavored simulation in the browser: agents pursue goals, societies emerge from local rules, and major beats are captured in an event feed and end-of-run chronicle.

  • Living map: terrain and resources drive movement and scarcity.
  • Emergent politics: settlements and factions form without a scripted story.
  • Readable outcomes: stats, notable figures, and trends when a run concludes.

Why it exists

The project explores how far you can push deterministic, inspectable simulation in a web app: clear tick order, separated engine and renderer, and room to grow through optional modules.

Design pillars
  • Determinism: same seed, same trajectory — great for tuning and demos.
  • Observability: click entities, filter events, follow arcs in the side panel.
  • Modularity: hooks at init, tick, day boundaries, decisions, and end checks.

What you actually do

Player role

You are less “god” and more director and archivist: steer time, pick seeds, toggle optional modules, and inspect whatever catches your eye.

  • Time controls: pause, resume, and run at 1×–8× speed.
  • Seeds: restart the same world or type a new seed to compare outcomes.
  • Demo presets: balanced, war-forward, or settlement-forward starting points.
  • Inspection: agents, settlements, factions, and world tabs with a filtered event feed.

Run it locally

  1. Clone the Epoch repo.
  2. npm install
  3. npm run dev and open the local URL Vite prints.
Build for static hosting
  1. npm run build
  2. Serve the dist folder from any static host (optional GitHub Pages project site).

Simulation logic stays in TypeScript; the canvas is a view layer on top.

Systems at a glance

High-level map of what is already implemented — not an exhaustive spec.

World and agents

  • World generation: hidden grid, clustered terrain and resources.
  • Agents: names, traits, avatars; needs loop (hunger, energy, health).
  • Decisions: utility-based scoring for what to do next.
  • Actions: movement, gathering, eating, resting.
Society and conflict
  • Settlements: founding, joining, deposits, lifecycle.
  • Factions: formation and dominance tracking.
  • Conflict: skirmishes with battle events in the log.
  • Migration: pressure-driven relocation to richer ground.

Presentation layer

  • Canvas rendering: world view with group colors for settlements and factions.
  • Hover labels: quick context on focused entities.
  • End screen: recap, notable figures, and simple trend charts.
  • Chronicle: sectioned narrative generated from structured events.
Example modules (toggleable)
  • Myths & Magic: flavor “omen” style events (placeholder gameplay).
  • Intergalactic Invaders: anomaly and meteor-style events (placeholder gameplay).

Modules register hooks so future work can add real effects without rewriting the core loop.

Engine rules

The simulation advances through an ordered set of systems each tick. Rendering reads state but does not drive gameplay rules — that boundary is intentional for testing, porting, and future features like replay.

Tooling you might use
  • Seed sweep script: scripts/seedSweep.ts for batch comparisons.
  • Demo mode: presenter-oriented overlay and preset wiring in the app.

Where it is headed

Near-term ideas

  • Tuning pass on scarcity, combat frequency, and late-game variety.
  • Module examples with small, visible gameplay effects beyond flavor text.
  • Chronicle prose variants tied to more event shapes.

Intentionally deferred

Deep diplomacy, scripted campaigns, and full timeline replay are out of scope for now — the current focus is a believable emergent loop you can study run to run.

Source of truth for day-to-day progress lives in the main repo’s PROJECT_STATUS.md (kept in sync with milestones).
